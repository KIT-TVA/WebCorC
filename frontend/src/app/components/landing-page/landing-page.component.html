Correctness-by-Construction (CbC) is an approach to incrementally create correct programs. CbC starts with an abstract Hoare triple {P} S {Q} consisting of a precondition P, an abstract statement S, and a postcondition Q. Hoare triples represent total correctness assertions that valuate only to true if, beginning from the precondition, the postcondition is met after executing the eventually defined program S. With CbC, the Hoare triple is successively refined using a set of refinement rules to a concrete implementation. Each application of refinement rules can be formally proven using their side conditions. CbC is implemented in the tool WebCorC. WebCorC is a graphical IDE to construct algorithms following the Correctness-by-Construction approach. It supports CbC developers to refine a program by a sequence of refinement steps and to verify the correctness of these refinement steps using the deductive verifier KeY.
